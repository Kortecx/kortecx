# API reference

The full SDK API references are **generated** from the SDK source and published
under [`/docs/api/`](pathname:///docs/api/):

- **TypeScript SDK** — [`/docs/api/typescript/`](pathname:///docs/api/typescript/),
  generated by [TypeDoc](https://typedoc.org).
- **Python SDK** — [`/docs/api/python/`](pathname:///docs/api/python/), generated
  by [pdoc](https://pdoc.dev).

:::note Generated, not hand-written
These references are produced by the docs pipeline and are not committed to
source control. They are empty in a fresh checkout until the pipeline runs. See
[`scripts/README.md`](https://github.com/Kortecx/kortecx/blob/main/docs/site/scripts/README.md)
for the exact commands and prerequisites.
:::

To populate them locally, from `docs/site/`:

```bash
npm run api:ts   # TypeDoc → static/api/typescript/
npm run api:py   # pdoc    → static/api/python/
```

Until the pipeline runs, the SDK surfaces are documented inline:

- [`bindings/python/README.md`](https://github.com/Kortecx/kortecx/blob/main/bindings/python/README.md)
- [`bindings/typescript/README.md`](https://github.com/Kortecx/kortecx/blob/main/bindings/typescript/README.md)
