kortecx live events

Live-tails a run over the R5 WebSocket bridge using @kortecx/sdk/web. This works against today's gateway (kx serve --ws-listen 127.0.0.1:50152) because the WS bridge re-skins the gRPC event stream as JSON. (Unary gRPC-web from the browser needs a gateway grpc-web/CORS handler — a follow-up gateway PR.)
